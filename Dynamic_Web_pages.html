Intro to JavaScript & the DOM

document.write · getElementById · innerHTML · alert

JavaScript is the programming language of the web. It runs inside the browser and lets you make web pages interactive — changing text, reacting to clicks, showing alerts, and much more.

What is the DOM?

The DOM (Document Object Model) is the browser's internal representation of your HTML page as a tree of objects. JavaScript can read and change any part of this tree — that's how a page "comes alive" after it loads.

HTML

The structure — headings, paragraphs, buttons, images.

JavaScript + DOM

The brain — reads the structure and changes it on the fly.

Key DOM Methods — Explained

Method / PropertyWhat it doesExample
document.write()Writes HTML text directly onto the page (during load)document.write("<h1>Hi</h1>")
document.getElementById(id)Finds and returns the HTML element with the given iddocument.getElementById("hello")
.innerHTMLGets or sets the HTML content inside an elementel.innerHTML = "<b>Bold!</b>"
alert()Shows a pop-up message box in the browseralert("Welcome!")
.style.propertyChanges the CSS style of an element directlyel.style.color = "red"

The First Project — Hello Webpage

HTML + JS<h1 id="hello">Hello</h1>
<script>
  // 1. Writes new HTML onto the page
  document.write("<h1>Welcome To My Webpage</h1>");

  // 2. Finds the <h1 id="hello"> element and changes its text
  document.getElementById("hello").innerHTML = "Hello. Whatsup!";

  // 3. Shows a popup box
  alert("Welcome");
</script>
⚠️ Caution

document.write() works fine during page load but if called after the page has loaded, it will erase the entire page content. Always prefer innerHTML for changing content after load.

Page Load Events — onload

body onload · hiding loading spinners

Sometimes you want JavaScript to run only after the whole page has finished loading — all images, videos, and elements. The onload event is perfect for this.

✅ Why use onload?

If your JS runs before the HTML elements exist, it can't find them — causing errors. onload guarantees everything is ready first.

💡 Real-world use

Show a "Loading…" spinner, then hide it once all images and videos on the page have fully downloaded.

HTML + JS<!-- The body calls display() once everything has loaded -->
<body onload="display()">

  <h1 id="loading">Loading...</h1>
  <img src="image.jpeg" height="200">
  <video src="video.mp4" controls></video>

  <script>
    function display() {
      // Hide the loading message once everything is ready
      document.getElementById("loading").style.display = "none";
    }
  </script>

</body>
🔎 How it works step-by-step
  • Browser starts reading the HTML top to bottom
  • The "Loading…" heading is shown immediately
  • Browser downloads the image and all videos
  • Once everything is ready, it fires the onload event
  • display() runs and hides the loading heading

⚠️ String + Number Gotcha

When you use + with a string and a number, JavaScript concatenates (joins) them instead of adding! Always convert with Number() or parseInt() first.

JS — Type Conversion Gotcha// Input from prompt() is always a STRING
var num1 = prompt("Enter number:");  // "5" (string!)
var num2 = prompt("Enter number:");  // "3" (string!)

console.log(num1 + num2);          // "53" ← WRONG! String concatenation

// Fix: convert to Number first
num1 = Number(num1);  // 5 (now a real number)
num2 = Number(num2);  // 3
console.log(num1 + num2);  // 8 ✅

Conditionals — if / else if / else

Decision-making in JavaScript

Conditionals let your program make decisions. Based on whether a condition is true or false, different blocks of code run.

JS — if / else if / elseif (condition1) {
  // runs when condition1 is true
} else if (condition2) {
  // runs when condition1 is false, condition2 is true
} else {
  // runs when ALL conditions above are false
}

Real Example — Even or Odd Checker

HTML + JS<input type="number" id="num">
<button onclick="checkEvenOdd()">Check</button>
<p id="result"></p>

<script>
  function checkEvenOdd() {
    var num = parseInt(document.getElementById("num").value);
    
    if (num % 2 === 0) {
      document.getElementById("result").innerHTML = num + " is Even";
    } else {
      document.getElementById("result").innerHTML = num + " is Odd";
    }
  }
</script>
💡 The Modulus Trick

num % 2 gives the remainder when dividing by 2. If the remainder is 0, the number is even. If it's 1, the number is odd. This works for any divisibility check!

06

setInterval & Timers

Running code repeatedly every N milliseconds

setInterval() is a built-in function that calls another function repeatedly, with a fixed time delay between each call. It's how we build clocks, timers, countdowns, and animations.

FunctionWhat it doesSyntax
setInterval()Runs a function every X milliseconds forever (until stopped)setInterval(fn, 1000)
clearInterval()Stops a running setIntervalclearInterval(timerID)
⏱️ Milliseconds

1000 milliseconds = 1 second. So setInterval(fn, 1000) calls fn once every second. setInterval(fn, 500) calls it twice per second.

JS — Basic Timer Patternvar time = 0;

function updateTimer() {
  time++;                                           // add 1 each second
  document.getElementById("timer").innerHTML = time;

  if (time === 10) {
    document.getElementById("clickBtn").style.display = "none";
    clearInterval(timerid);    // ← STOP the timer
  }
}

var timerid = setInterval(updateTimer, 1000); // start: every 1 second
⚠️ Always store the timer ID!

Save the return value of setInterval() into a variable — that's how you can stop it later with clearInterval(id). Without the ID, the timer runs forever.

JavaScript has a built-in Date class. When you write new Date(), you create an object representing the current date and time. You then call methods on it to extract specific parts.

Live Digital Clock

JSfunction clock() {
  var hh = new Date().getHours();

If the hour is 9, you might want to show "09" not "9". Use this trick: if (hh < 10) { hh = '0' + hh; }

for Loops

Repeating code a set number of times

A for loop runs a block of code a specific number of times. It's perfect for generating repeated content — like a multiplication table.

JS — for loop anatomyfor (var i = 1; i <= 10; i++) {
  // runs 10 times: i = 1, 2, 3, ..., 10
  console.log(i);
}
//  ↑ start  ↑ condition  ↑ increment

Project — Multiplication Table Generator

JSfunction mult() {
  var n = document.getElementById("num").value;
  var r = "";   // empty string to build result

  for (var a = 1; a <= 10; a++) {
    var c = n * a;
    r += n + " x " + a + " = " + c + "<br>";
    // += means "add to the existing string"
  }

  document.getElementById("para").innerHTML = r;
}
🔎 Why no parseInt needed here?

When you multiply a string by a number (e.g., "5" * 3), JavaScript automatically converts the string to a number. This is called implicit type coercion. It works for *, /, -, but NOT for + (which does string joining instead).

Project — Light Bulb Toggle

Boolean flag · toggling state · changing images

This project teaches a fundamental pattern: using a boolean variable as a flag to track the current state (on or off), and toggling it each time the user clicks.

JSvar isOn = false;   // track state: is bulb on?

function toggleBulb() {
  const bulb = document.getElementById("bulb");

  if (isOn == true) {
    bulb.src = "pic_bulboff.gif";   // currently on → turn off

💡 The Toggle Pattern

eval() takes a string like "12 + 5 * 3" and evaluates it as actual JavaScript math, returning the result. It's very powerful but should only be used in controlled environments like this — not with user-typed input in production apps.

Projects — Dice Roller & Color Changer

Math.random() · Math.floor() · generating random numbers

Math.random() is JavaScript's tool for generating random numbers. Understanding how to use it is key to building games and dynamic effects.

🔎 How Math.random() works

Math.random() returns a decimal between 0 (inclusive) and 1 (exclusive), like 0.7382.... To get a random integer in a range, you scale and floor it.

JS — Random number recipe// Random integer from 1 to 6 (dice roll):
Math.floor(Math.random() * 6) + 1
//  Math.random() → 0.0 to 0.9999...
//  * 6           → 0.0 to 5.9999...
//  Math.floor()  → 0, 1, 2, 3, 4, or 5
//  + 1           → 1, 2, 3, 4, 5, or 6  ✅

Dice Roller

JSfunction rollDice() {

  document.getElementById("dice").innerText = "🎲 " + num;
}

Background Color Changer

Project — Guess the Color Game

RGB colors · dropdown select · attempts counter · reset

Five dropdown options are each given a random background color using rgb(r, g, b). One of them is the "correct" color. The player picks options until they find the right one.

JS — Core game logicvar correctColor = "";
var chances = 0;

function setColors() {
  for (var i = 1; i <= 5; i++) {
    var r = Math.floor(Math.random() * 256); // 0–255
    var g = Math.floor(Math.random() * 256);
    var b = Math.floor(Math.random() * 256);
    var rgb = "rgb(" + r + ", " + g + ", " + b + ")";
    
    document.getElementById("option" + i).style.backgroundColor = rgb;
  }

  // Randomly pick one option as the correct answer
  var randomIndex = Math.floor(Math.random() * 5) + 1;
  correctColor = document.getElementById("option" + randomIndex).style.backgroundColor;
}

function check() {
  var selectedId    = document.getElementById("color").value;
  var selectedColor = document.getElementById(selectedId).style.backgroundColor;
  
  chances++;

  if (selectedColor == correctColor) {
    document.getElementById("result").innerText = "✅ Correct! You guessed in " + chances + " chances!";
    document.getElementById("color").style.display = "none";
  } else {
    document.getElementById("result").innerText = "❌ Incorrect! Try again.";
  }
}

setColors(); // start game when page loads
💡 Why 256?

Colors in RGB have 256 possible values per channel (0–255). Using Math.floor(Math.random() * 256) gives you a random integer from 0 to 255 — exactly the range needed for a valid RGB color!

Canvas Basics

HTML canvas element · 2D context · shapes · colors

The HTML <canvas> element is like a blank drawing board inside the browser. JavaScript draws on it using a special object called the 2D rendering context.

HTML + JS — Canvas Setup<!-- Step 1: Create the canvas in HTML -->
<canvas id="myCanvas" width="400" height="300"
        style="border:1px solid black;"></canvas>

<script>
  // Step 2: Get reference to canvas element
  const canvas = document.getElementById("myCanvas");

  // Step 3: Get the 2D drawing context (the "pen")
  const ctx = canvas.getContext("2d");

  // Step 4: Draw a filled rectangle
  ctx.fillStyle = "blue";
  ctx.fillRect(50, 50, 100, 100); // x, y, width, height

  // Step 5: Draw a filled circle
  ctx.beginPath();
  ctx.arc(200, 150, 40, 0, Math.PI * 2); // x, y, radius, start, end angle
  ctx.fillStyle = "red";
  ctx.fill();
  ctx.closePath();
</script>

🔎 Canvas Coordinate System

The canvas origin (0,0) is the top-left corner. X increases to the right, Y increases downward. So fillRect(50, 100, 80, 60) draws a rectangle starting 50px from the left and 100px from the top.

💡 Drawing a Triangle

Canvas has no built-in triangle shape. You draw one by using moveTo() to go to the first point, then lineTo() for the second and third points, and closePath() to connect back to the start — forming a closed triangle path you can fill.

Canvas Animation

requestAnimationFrame · moving circle · bouncing box

Animation = draw → erase → draw at new position → repeat. We use requestAnimationFrame() which tells the browser to run our drawing function before the next screen refresh — producing silky-smooth ~60fps animation.

  // ... draw shapes at new positions ...

  requestAnimationFrame(draw); // call draw() again next frame
}

Project 1 — Moving Circle (Bouncing Left–Right)

JSlet x = 50;
let speed = 2;

function draw() {
  ctx.clearRect(0, 0, canvas.width, canvas.height);

  ctx.beginPath();
  ctx.arc(x, 150, 20, 0, Math.PI * 2);
  ctx.fillStyle = "green";
  ctx.fill();

  x += speed;   // move right by 'speed' pixels each frame

  // Bounce: if hitting right or left edge, reverse direction
  if (x + 20 > canvas.width || x - 20 < 0) {
    speed = -speed;  // flip direction!
  }

  requestAnimationFrame(draw);
}

draw();

Project 2 — Bouncing Box (2D Bounce)

JSlet x = 10, y = 10;   // position
let dx = 3, dy = 2;   // velocity (speed + direction)

function draw() {
  ctx.clearRect(0, 0, canvas.width, canvas.height);

  ctx.fillStyle = "blue";
  ctx.fillRect(x, y, 50, 50);

  x += dx;  y += dy;  // move each frame

  // Bounce off left/right walls
  if (x + 50 > canvas.width  || x < 0) dx = -dx;
  // Bounce off top/bottom walls
  if (y + 50 > canvas.height || y < 0) dy = -dy;

  requestAnimationFrame(draw);
}

draw();
🚀 The Key Animation Ideas
  • Position variables (x, y) track where the object is
  • Velocity variables (dx, dy) control how fast and in which direction it moves
  • Bouncing is done by reversing velocity (dx = -dx) when hitting a wall
  • clearRect() erases the old frame before drawing the new one
  • requestAnimationFrame() keeps the loop running smoothly (~60 times per second)
