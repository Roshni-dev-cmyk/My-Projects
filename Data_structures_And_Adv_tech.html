1. Python Lists

Store many items together in one single variable

🤔 What is a List?

A list is a collection that holds many items inside one variable. Items are written inside [ ], separated by commas. Lists are:

  • Ordered — items stay in the position you placed them
  • Changeable — you can add, remove, or edit items at any time
  • Allow duplicates — the same item can appear more than once
  • Can hold mixed types — numbers, strings, even other lists!
🌍 Real-Life Examples of Lists
  • 🎒 Your school bag — it holds a book, pencil, bottle, eraser all in one place. A Python list is exactly like that — one variable holding many items.
  • 📱 Your contact list on a phone — all phone numbers stored together, in the order you added them.
  • 🛒 A shopping list — "milk, eggs, bread, butter" — items in a specific order you can add to or cross off.
  • 🏆 A scoreboard — stores scores of all players: [95, 82, 78, 91, 60]

📌 1.1 List Methods

🔧 What are Methods?

Methods are built-in actions you can perform on a list. Think of them as superpowers your list already has! You use them by writing listname.method().

🌍 Classroom Attendance Analogy

Imagine a class register (attendance list). Think of each list method as an action a teacher might do with it:

  • 📝 append = A new student joins and sits at the very last seat
  • 📌 insert = A transfer student joins and is assigned a specific seat number
  • 📋 extend = An entire new group of students joins the class at once
  • ❌ pop(i) = A student at seat number i leaves the class
  • 🗑️ remove(x) = A specific named student leaves (you say the name, not the seat)
  • 🧹 clear = The whole class is emptied — fresh start!

📌 1.3 List Comprehension

✨ What is List Comprehension?

List comprehension is a shortcut for creating a new list from an existing one — written in a single line instead of 4-5 lines of code. It's one of Python's most powerful and elegant features!

💡 The Formula

new_list = [expression for item in old_list if condition]

🌍 Real-Life Examples of List Comprehension
  • 🍎 Sorting a fruit basket — You have a big basket of mixed fruits. You want only the apples in a new basket. List comprehension is like a magic hand that picks ONLY apples in one swift motion!
  • 🏫 Filtering students — Out of a class list, pick only students whose names start with the letter "N". Instead of reading every name one by one and writing it down, list comprehension does it instantly.
  • 📊 Creating a marks report — From a list of marks, create a new list of only the students who scored above 80 — in one line of code.
  • 🔢 Squares table — Instead of writing a whole loop to find squares of 1 to 10, list comprehension does it: [x**2 for x in range(1,11)]

# ── Old way: find even numbers using a for loop

2. Python Stack

Last In, First Out (LIFO) — like a stack of plates!

🤔 What is a Stack?

A stack is a special list where you can only add or remove items from the TOP. The rule is called LIFO — Last In, First Out. Whatever you put in last is the first thing to come out — like a stack of plates or a pile of books.

  • Push = Add an item to the TOP (uses append())
  • Pop = Remove the TOP item (uses pop())
  • Peek = Just look at the top item without removing it (uses stack[-1])
  • isEmpty = Check if the stack has nothing in it
  • isFull = Check if the stack has reached its maximum size
🌍 Where Do We See Stacks in Real Life?
  • 🍽️ Stack of plates in a cafeteria — You always pick up the plate from the TOP. The last plate placed by the kitchen staff is the first one you take. You can't grab a plate from the bottom without removing the top ones first.
  • ↩️ Ctrl+Z (Undo) in any app — Every action you do is "pushed" onto a stack. When you press Undo, the most recent action is "popped" off and reversed. That's why Undo always reverses your very last action!
  • 🌐 Browser Back button — Every page you visit is pushed onto a stack. Pressing Back pops the current page and returns you to the previous one.
  • 📚 A pile of homework — You put new sheets on top. When the teacher checks them, she picks from the top — the last paper added gets checked first.
  • 🔄 Function calls in programs — When Python runs a function inside a function, it stacks them up. When the inner function finishes, it's popped off and the outer one continues.
🧠 Stack vs Normal List

A normal list lets you add or remove items from anywhere. A stack is a restricted list — you can ONLY touch the TOP item. Imagine a can of tennis balls — you can only take the top ball out, and you can only drop new balls in from the top!

📌 2c. Reverse String & Number Using Stack

🔄 How Does Reversing Work with a Stack?

Push each character (or digit) onto the stack one by one → then pop them all off. Because of LIFO, the last character pushed comes out first — giving you the reversed string!

🌍 The Sticky Notes Trick

Imagine writing each letter of "HELLO" on separate sticky notes and sticking them on a wall one on top of the other:

  • Push: H → E → L → L → O (O is now on top)
  • Pop: O → L → L → E → H
  • Result: "OLLEH" — reversed!

This same logic is used when you press Backspace on a keyboard — it removes characters in reverse order of how you typed them!

3.reverse_string.py
def rev(st):
    result = ""
    stack  = []
    for i in st:
        stack.append(i)          # push each character
    while len(stack) != 0:
        result = result + stack.pop()   # pop from top → reversed!
    print(result)

while True:
    ans = input("Reverse a string? (yes/no): ").lower()
    if ans == "yes":
        st = input("Enter string: ")
        rev(st)
    else:
        break
4.reverse_number.py
def rev(num):
    stack = []
    while num != 0:
        last = num % 10        # get the last digit
        stack.append(last)     # push it onto stack
        num  = num // 10      # remove last digit from number
    i      = 1
    result = 0
    while len(stack) != 0:
        result = result + (stack.pop() * i)
        i = i * 10
    print(result)

num = int(input("Enter a number: "))
rev(num)   # e.g. 12345 → 54321

📌 2d. Recursion

🌀 What is Recursion?

Recursion is when a function calls itself! It breaks a big problem into a smaller version of the same problem, and keeps doing that until it reaches a base case — the stopping condition that prevents it from going on forever.

💡 Two Must-Haves in Recursion

1. Base case — the condition to STOP (e.g., if num == 1: return 1)
2. Recursive call — the function calling itself with a smaller value

🌍 Real-Life Examples of Recursion
  • 📢 Whisper chain in class — A teacher tells one student, who tells the next, who tells the next, until the message reaches the end. Each person is a "call", the last person is the "base case".
  • 🪆 Russian nesting dolls (Matryoshka) — Each doll contains a smaller version of itself. You keep opening until you reach the tiniest doll (base case).
  • 🪞 Mirrors facing each other — When two mirrors face each other, you see a reflection inside a reflection inside a reflection… it eventually gets too small to see (base case).
  • 📂 Folders inside folders — When your computer searches inside a folder, it finds sub-folders and searches inside those too — this is recursive searching!
1.factorial.py — Factorial using Recursion
# Factorial: 5! = 5 × 4 × 3 × 2 × 1 = 120
# Think of it as: fact(5) = 5 × fact(4)
#                           4 × fact(3)
#                               3 × fact(2)
#                                   2 × fact(1)  ← base case!

def fact(num):
    if num == 1:                    # BASE CASE — stop here
        return 1
    else:
        return num * fact(num - 1)  # RECURSIVE CALL

num = int(input("Enter a number: "))
if num < 0:
    print("Factorial of negative numbers doesn't exist.")
elif num == 0:
    print("Factorial of 0 is 1")
else:
    print(fact(num))

# Fibonacci series: 0, 1, 1, 2, 3, 5, 8, 13, 21 ...

📌 2e. Stack Using Tuple — Bracket Checker

🔍 What is a Tuple?

A tuple is like a list but immutable — once created, it cannot be changed. We use tuples when we want to store data that should never be accidentally modified during the program. Here we store the bracket types in tuples because those values should never change.

🌍 The Bracket Matching Story

In maths you write expressions like: (a + b) × [c - (d / e)]. Every opening bracket must have a matching closing bracket of the same type. A stack is the perfect tool to check this!

  • Every time you see (, [, or { → PUSH it onto the stack (remember it!)
  • Every time you see ), ], or } → CHECK if the top of the stack is the matching opener. If yes → POP it. If no → expression is unbalanced!
  • At the end, if the stack is empty → all brackets matched ✅. If not → something's wrong ❌

This is exactly how code editors like VS Code highlight bracket errors in red!

c.Python-stack using tuple.py — Bracket Checker
openP  = ('{', '[', '(')   # tuple — can't be changed accidentally
closeP = ('}', ']', ')')
stack  = []

def check(expr):
    for c in expr:
        if c in openP:
            stack.append(c)           # push opening bracket
            print(stack)
        elif c in closeP:
            index = closeP.index(c)
            if openP[index] == stack[-1]:
                stack.pop()           # matched! pop it
                print(stack)
    if len(stack) == 0:
        print("✅ Balanced!")
    else:
        print("❌ Not balanced!")

string = input("Enter the expression: ")
check(string)
# Try: {a+b-(c*d)/[a+(d^c)]}  → Balanced ✅
# Try: (a+b]                   → Not balanced ❌

3. Python Queue

First In, First Out (FIFO) — like a line at a ticket counter!

🤔 What is a Queue?

A queue is a list where items are added at the BACK and removed from the FRONT. This is called FIFO — First In, First Out. The first person to join the queue is the first to be served — fair and orderly!

  • Enqueue = Join the back of the queue (append())
  • Dequeue = Serve the person at the front (pop(0))
  • Peek = See who is next at the front (queue[0])
  • isEmpty / isFull = Check the queue's status
🌍 Where Do We See Queues in Real Life?
  • 🎟️ Movie ticket counter — People stand in a line. Whoever came first gets the ticket first. No cutting in line!
  • 🍔 Fast food restaurant — Orders are taken and fulfilled in the order they were placed. The first order placed is the first to be made.
  • 🖨️ Printer queue — When multiple students send documents to print, the printer prints them in the exact order they were sent. If your document was sent first, it prints first.
  • 🏥 Hospital emergency waiting room — Patients are added to a queue. The one who arrived earliest gets attended first (assuming similar urgency).
  • 📞 Customer care call center — "You are caller number 4 in the queue." The person who called first gets connected first!
  • 🚗 Car wash — Cars line up, and the first car in gets washed first. New cars join at the back.
🧠 Stack vs Queue — The Key Difference

Stack = A can of Pringles chips. You add and remove from the same end (top). Last chip in = first chip out.

Queue = A tube of toothpaste. You squeeze from one end (front) and add paste from the other end (back). First paste in = first paste out.

📌 3.3 Palindrome Checker

🔁 What is a Palindrome?

A palindrome is a word (or number) that reads the same forwards and backwards!

  • ✅ madam → m-a-d-a-m backwards is still m-a-d-a-m
  • ✅ racecar, level, noon, refer, civic
  • ❌ python, hello, code — not palindromes
🌍 How We Use a Queue to Check Palindromes

Put each letter of the word into a queue. Then dequeue from the front one by one, but instead of adding each dequeued letter to the end of a new string, add it to the front. This reverses the word. If the reversed word equals the original → it's a palindrome!

This is useful in real applications like checking if a DNA strand reads the same in both directions, or validating symmetric patterns in data!

3.palindrome_checker.py
queue = []
word  = input("Enter a word: ")

# Add each character to the queue
for i in word:
    queue.append(i)

# Dequeue from front, add to FRONT of new string → reverses it
rev = ""
while len(queue) > 0:
    item = queue.pop(0)
    rev  = item + rev

if rev == word:
    print("✅ It IS a palindrome!")
else:
    print("❌ Not a palindrome.")

🌍 The Cinema Hall Scenario

It's a Friday evening and a new superhero film just released! 🎬

  • The hall has only 100 seats total.
  • Customers arrive and join the queue with the number of tickets they want.

4.cnmaqu.py — Cinema Ticket Queue
import time
import random

queue         = []
hall_capacity = 100

def join_queue(customer, tickets):
    total = sum(t for _, t in queue)
    if total + tickets <= hall_capacity:
        print(f"🎟️ {customer} joined for {tickets} ticket(s)")
        queue.append((customer, tickets))
    else:
        print(f"❌ Sorry {customer}! Not enough seats.")

def serve_customer():
    if queue:
        customer, tickets = queue.pop(0)
        print(f"💳 {customer} is buying {tickets} ticket(s)...")
        time.sleep(random.randint(1, 3))
        print(f"✅ {customer} got their tickets!")
    else:
        print("Queue is empty.")

customers = ["Alice", "Bob", "Charlie", "David", "Eve"]
for customer in customers:
    tickets = int(input(f"How many tickets for {customer}? "))
    join_queue(customer, tickets)

print("\n🎬 Serving customers...\n")
while queue:
    serve_customer()
print("All customers served!")

🔄 3.5 Circular Queue — Hot Potato & Traffic Signal

♻️ What is a Circular Queue?

In a circular queue, items cycle in a loop — the back connects to the front endlessly. Unlike a normal queue where items leave permanently, in a circular queue items keep going around and around until something eliminates them.

🌍 Real-Life Examples of Circular Queues
  • 🎠 Merry-go-round — Kids go round and round. The same seats keep cycling. No beginning or end!
  • 🥔 Hot Potato game — Children stand in a circle. A "potato" (ball) is passed around. When the music stops, whoever holds the potato is out. The circle keeps shrinking until one winner remains.
  • 🚦 Traffic signal — Red → Yellow → Green → Red → Yellow → Green… it cycles forever, serving vehicles in a round-robin pattern.
  • 💻 CPU scheduling — Modern computers use circular queues to give every program a small "turn" of CPU time in rotation — so no single program hogs the processor.
  • 🎵 Music playlist on repeat — When a playlist ends and loops back to the first song, that's circular queue logic!
1.1.hotpotato.py — Hot Potato Game
import random

def playgame():
    while len(q) > 1:
        num = random.randint(1, 10)  # random number of passes

4. File Handling

Save data permanently — even after the program closes!

📝 Why Do We Need File Handling?

When a Python program closes, all data stored in variables is lost forever — because computer RAM (memory) is temporary, like a whiteboard that gets erased when you leave the room. To save data permanently, we write it to files stored on the hard drive. Files are like notebooks that stay on your desk even after you leave school.

💡 3 Steps of File Handling — Always in This Order!

1. Open the file → 2. Read or Write data → 3. Close the file

🌍 Real-Life Examples of File Handling
  • 🎮 Video game save files — You press "Save Game" and your character's level, items, and progress are written to a file. Next time you open the game, Python reads that file and restores exactly where you left off. Without file handling, every time you close a game you'd start from scratch!
  • 📒 School attendance register — The teacher writes each student's attendance in a register (file). Next week they open the same register to check last week's data.
  • 📧 Email drafts — When you start writing an email and close the app, your draft is saved as a file. When you reopen it, it reads the file and shows your draft.
  • 📸 Photo gallery — Every photo you take is saved as an image file on your phone's storage. Without file handling, photos would disappear the moment you close the camera app!
  • 🏦 Bank transaction records — Every time you deposit or withdraw money, the bank's system writes a record to a file. This is how your bank statement is generated later.

📌 4.4 Image Files — Binary Mode

🖼️ How Are Images Stored in Computers?

Computers only understand 0s and 1s (binary). Text files store characters as text, but images, audio, and video files store data as raw binary. We need special modes ("rb", "ab") to handle these files correctly.

💡 Bits and Bytes

8 bits = 1 byte. One byte stores one character of text. A small image can have millions of bytes! A bytearray is a collection of bytes — like a list of 0-255 numbers.

🌍 Secret Messages in Images (Steganography!)

This is a real technique called steganography — hiding secret messages inside image files! By appending text in binary mode to a PNG image, you can embed a hidden message that ordinary people won't see when they open the image in a photo viewer. Only someone who knows to read the binary data with Python can discover the message. Spies and hackers use this in movies and in real cybersecurity!

5. Exception Handling

Handle errors gracefully so your program never crashes unexpectedly!

💥 What is an Exception?

Even when a program is written with correct syntax, things can go wrong when it's actually running — because users do unexpected things! An exception is an error that happens during execution. Without exception handling, your program simply crashes and stops.

  • Syntax Errors — Wrong grammar in code. Python catches these before running.
  • Runtime Errors / Exceptions — Things go wrong while the program is running.
🌍 Real-Life Examples of Exceptions
  • 🧮 Calculator dividing by zero — If you ask a calculator to divide 10 by 0, it shows an error instead of crashing. That's exception handling!
  • 🏧 ATM machine — If you type letters instead of a PIN number, the ATM doesn't crash — it says "Invalid input, please enter numbers only." The ATM handles the ValueError gracefully.
  • 🔑 Wrong password — If you type the wrong password too many times, the app doesn't crash — it says "Too many attempts, account locked." That's a raised exception.
  • 📁 Opening a deleted file — If a program tries to open a file that the user already deleted, instead of crashing it says "File not found." That's handling an IOError.
  • 🌐 Website error pages — When a website can't find a page, instead of a scary crash it shows "404 – Page Not Found." That's exception handling in web apps!

📌 5.1 Types of Errors

🧠 Syntax Error vs Runtime Error

Syntax Error = Like writing a sentence with missing punctuation that your English teacher catches before you hand it in — "I wants to eat" → grammar is wrong before it's even submitted.

Runtime Error = Like giving a perfectly written recipe to the chef, but they try to add an ingredient that's not in the kitchen. The recipe was fine — the problem happened while executing it.

6. Python JSON

A universal language for sharing data between apps and websites!

🤔 What is JSON?

JSON (JavaScript Object Notation) is a standard format for storing and sharing structured data as text. It's not a Python thing — it works across ALL programming languages (Python, Java, C++, JavaScript, etc.). This makes it the universal "language" that different apps use to talk to each other!

  • JSON data looks very similar to a Python dictionary: keys and values inside { }
  • JSON is always stored as a string — even if it looks like a dictionary
  • Python has a built-in json module to easily convert between Python and JSON
🌍 Real-Life Examples of JSON
  • 🌤️ Weather app — When you open a weather app, it asks a server "What's the weather in Mumbai?" The server sends back JSON data:
    {"city": "Mumbai", "temp": 32, "condition": "sunny", "humidity": 70}
    The app then reads (parses) this JSON and displays it beautifully!
  • 📱 Instagram / social media — When you scroll your feed, your phone sends a request to Instagram's servers. The server responds with JSON containing your posts, likes, comments, profile pictures. Your app displays them on screen.
  • 🛍️ Online shopping — When you search for a product on Amazon, their server sends JSON: {"product": "shoes", "price": 1499, "inStock": true, "rating": 4.5}
  • 🎮 Game settings / save files — Many modern games store player settings and save data in JSON files: {"player": "Rohan", "level": 12, "health": 85, "coins": 4500}
  • 🗺️ Google Maps — Location data, routes, and business info are all exchanged as JSON between your phone and Google's servers.

     'married': True, 'age': 32}

print(type(s))     # <class 'str'>  ← it's a string now!

s = '{"name": "Bob", "languages": ["English", "French"], "age": 32}'
d = json.loads(s)
print(d)           # {'name': 'Bob', 'languages': [...], 'age': 32}
print(type(d))     # <class 'dict'>  ← Python dict again!

# Now you can use it like any normal dictionary
print(d["languages"])  # ['English', 'French']
print(d["name"])        # Bob

7. Interactive Dictionary Project

📖 What Does This Project Do?

This is a fully working English dictionary program that:

  • Loads thousands of English words and their meanings from a data.json file
  • Lets the user search for any word and see all its meanings
  • If the word isn't found, it suggests similar words (like autocorrect!) using Python's difflib module
  • Keeps running until the user says "no" — a complete, interactive program!
🌍 Real-Life Connection — How Google Search Autocorrect Works

Have you ever typed something wrong in Google and it still found what you were looking for? Like typing "pythn" and Google says "Showing results for: python"? That's fuzzy matching — and this project does the same thing!

  • Type "catt" → program suggests "cat" ✅
  • Type "hous" → program suggests "house" ✅
  • Type "serentipity" → program suggests "serendipity" ✅

Python's difflib.get_close_matches() compares your input word to ALL words in the dictionary and returns those that are most similar in letter patterns. This is the same technology used in spell checkers, search engines, and DNA sequence matching in biology!

🔬 How difflib Works

difflib.get_close_matches(word, possibilities) measures how similar two strings are by comparing their sequences of characters. It returns a list of the closest matching words from your dictionary keys. You can control how many suggestions to return and how close the match must be.

🧠 The Similarity Score Idea

"cat" and "catt" are 75% similar (3 of 4 characters match). "cat" and "xyz" are 0% similar. difflib calculates this score for your input word vs every word in the dictionary, and returns those above a threshold. It's like finding the closest rhyme to a word!

📌 7.1 The Complete Project Code

inter_dict_final.py — Interactive Dictionary
import json
import difflib

# Load dictionary data from JSON file
# The JSON has structure: {"word": ["meaning1", "meaning2", ...], ...}
f    = open('data.json')
data = json.load(f)    # converts the JSON file → Python dictionary

while True:
    answer = input("Do you want to find the meaning of a word? : ")

    if answer == 'yes':
        word = input('Enter the word: ')

        if word in data.keys():
            # ✅ Exact match found!
            meanings = data[word]
            i = 1
            print(f'\nAll meanings of "{word}":')
            for m in meanings:
                print(f"  {i}. {m}")
                i += 1

        else:
            # ❌ Exact match not found — try fuzzy matching!
            matchingwords = difflib.get_close_matches(word, data.keys())

            for x in matchingwords:
                answer = input(f'Did you mean "{x}"? (yes/no): ')
                if answer == 'yes':
                    meanings = data[x]
                    i = 1
                    print('\nAll meanings:')
                    for m in meanings:
                        print(f"  {i}. {m}")
                        i += 1
                    break   # stop once we found and showed the meaning
            else:
                print('❌ No such word exists in the dictionary.')
    else:
        break
