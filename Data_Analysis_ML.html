📋 DataFrames & Series
Pandas has two core data structures. Master these two and everything else will click naturally.
THINK OF IT LIKE THIS
Imagine the IPL scorecard: a big spreadsheet with player names as rows and columns like Runs, Wickets, Balls, Average, Strike Rate. That entire spreadsheet is a DataFrame. If you click on just the "Runs" column and drag it out — that's a Series. Pandas DataFrames are Excel-on-steroids: they can hold 10 million rows, do calculations in milliseconds, and never crash.
📋 DataFrame — The Table
A 2D table with rows and columns. Each column can hold different data types (numbers, text, dates). This is the backbone of all data analysis.
python
import pandas as pd

📌 Series — One Column
A 1D labeled list. Pulling one column from a DataFrame gives you a Series. It has an index and values — like a Python dictionary but faster.
python

Single vs Double brackets — a critical difference: df["col"] returns a Series (no table structure). df[["col"]] returns a DataFrame (still a table with one column). The extra brackets preserve the table shape — very important when passing data into ML models which expect a 2D table as input!

Essential exploration methods for any new DataFrame: df.head(5) — first 5 rows  ·  df.tail(3) — last 3 rows  ·  df.shape — (rows, columns)  ·  df.columns — column names  ·  df.dtypes — data type of each column  ·  df.info() — complete summary including nulls  ·  df.describe() — stats for all numeric columns (mean, std, min, max, quartiles)

📍 .loc — Label-based
Uses row/column names (labels). End of range is INCLUDED.

df.loc[2:5] → rows 2, 3, 4, 5 included
df.loc["Alice", "Score"] → exact cell by name
🔢 .iloc — Position-based
Uses row/column numbers (0, 1, 2…). End of range is EXCLUDED (like Python slices).

df.iloc[2:5] → rows 2, 3, 4 only
df.iloc[0, 1] → row 0, column index 1
FILTERING LIKE A ZOMATO DATA ANALYST
Imagine you're working at Zomato and have a DataFrame with restaurant data. Here's how you'd answer real business questions:
  • Show only top-rated restaurants: df[df["rating"] > 4.5]
  • Restaurants in Mumbai OR Delhi: df[(df["city"]=="Mumbai") | (df["city"]=="Delhi")]
  • Chinese restaurants that deliver: df[(df["cuisine"]=="Chinese") & (df["delivery"]==True)]
  • Search in 3 cities at once: df[df["city"].isin(["Mumbai","Pune","Bangalore"])]
  • Restaurants with "biryani" in name: df[df["name"].str.contains("Biryani", case=False)]
⚠️
The parentheses mistake everyone makes: When combining conditions with & or |, ALWAYS wrap each condition in parentheses.
❌ Wrong: df[df["a"]>5 & df["b"]=="x"]
✅ Correct: df[(df["a"]>5) & (df["b"]=="x")]
Python's operator precedence will cause a cryptic error without the parentheses.

GROUPBY IN ACTION — BCCI MATCH DATA
Imagine you have a DataFrame of every IPL match with columns: Player, Team, Runs, Wickets, Match.
  • Total runs by each team: df.groupby("Team")["Runs"].sum()
  • Average runs per player: df.groupby("Player")["Runs"].mean()
  • Number of players per team: df.groupby("Team")["Player"].count()
  • Top scorer per team: df.groupby("Team")["Runs"].max()
This is exactly how sports analytics dashboards are built. GroupBy is the secret weapon behind every leaderboard, comparison table, and trend report.
🧠
axis=0 vs axis=1 — which direction are we going? axis=0 (default) works down each column — e.g., get average of all Math scores in one column. axis=1 works across each row — e.g., get average of one student's three subject scores. Memory trick: axis=0 = down the rows (like a 0 is a tall oval ⬇️), axis=1 = across columns (like a 1 is a horizontal stick ➡️).

What we're doing Store student data in plain Python lists, compute basic stats, then draw a bar chart. This is the classic 'hello world' of data visualization — the exact same pattern is used in school report systems and sports dashboards.
python
import matplotlib.pyplot as plt

# Store student data in lists
students    = ["Alice", "Bob", "Charlie", "Diana", "Eve"]
math_scores = [88, 76, 93, 85, 91]

New concept: df.set_index() Normally DataFrame rows are numbered 0,1,2… Setting a column as the index means rows are now labeled by student name. This makes Pandas use the names automatically as the X-axis when you call df.plot() — you don't need to pass X separately.
ℹ️
axis=1 vs axis=0 axis=1 calculates across each row — so each student's three scores get averaged together. axis=0 calculates down each column — so all students' Math scores get averaged together. Think of it as: axis=1 goes left→right (horizontal), axis=0 goes top→bottom (vertical).
python
import pandas as pd
import matplotlib.pyplot as plt

data = {
    "Student": ["Alice", "Bob", "Charlie", "Diana", "Eve"],
    "Math":    [88, 76, 93, 85, 91],
    "Science": [92, 80, 89, 88, 95],
    "English": [85, 79, 90, 84, 87]
}
df = pd.DataFrame(data)

# Add average column — axis=1 means average ACROSS columns (per row)
df["Average"] = df[["Math", "Science", "English"]].mean(axis=1)

# Average PER SUBJECT (across all students)
avg_per_subject = df[["Math", "Science", "English"]].mean()
print(avg_per_subject)

# Set Student column as index → rows now labeled by name
df.set_index("Student", inplace=True)

# Plot grouped bar chart
df[["Math", "Science", "English"]].plot(
    kind="bar",
    figsize=(8, 6),
    color=['skyblue', 'salmon', 'lightgreen']
)
plt.title("Scores of Students in Multiple Subjects")
plt.xlabel("Students")
plt.ylabel("Scores")
plt.ylim(0, 100)
plt.xticks(rotation=45)   # tilt labels 45° so they don't overlap
plt.legend(title="Subjects")
plt.show()
💡
The .plot() shortcut After df.set_index('Student'), calling df[['Math','Science','English']].plot(kind='bar') automatically uses student names as x-ticks and creates grouped bars for each column. Specify figsize=(8,6) to make it wide enough.

Why use a custom index? Setting employee names as the index lets you write df.loc['Bob'] instead of df.loc[1]. It's like giving rows meaningful names instead of numbers — much more readable and less likely to cause off-by-one errors if rows get deleted or reordered.
ℹ️
.loc vs .iloc — a memory trick .loc = Label-based (use the name). .iloc = Integer position (use the number). When you do df.loc[2:5] the end 5 is included. When you do df.iloc[2:5] the end 5 is excluded — just like regular Python list slicing.
python
import pandas as pd

data = {
    "Salary":     [50000, 60000, 75000, 52000],
    "Department": ["HR", "IT", "Finance", "IT"],
    "Experience": [2, 5, 8, 3]
}
# Custom row index using employee names!
df = pd.DataFrame(data, index=["Alice", "Bob", "Charlie", "David"])

# ── Column selection ──────────────────────────
df["Salary"]                      # → Series
df[["Salary", "Experience"]]      # → DataFrame

# ── Row selection with .loc (label-based) ─────
df.loc["Bob"]                     # → Series (one employee)
df.loc[["Bob"]]                   # → DataFrame (one row)
df.loc[["Alice", "Charlie"]]     # → DataFrame (two rows)

# ── Specific cells with .loc ──────────────────
df.loc["Charlie", "Department"]       # → "Finance"
df.loc["David", ["Salary", "Experience"]]

# ── .iloc (position-based) ───────────────────
df.iloc[0, 0]    # 1st row, 1st col → Alice's Salary
df.iloc[1:3, :]  # rows 1-2 (Bob + Charlie), all columns
df.iloc[:, 1:3] # all rows, columns 1-2

# ── Filter employees with 5+ years experience ─
df[df["Experience"] >= 5]

🎯
New method: .mode() .mode() returns the most frequently occurring value — the 'fashion' of statistics (most fashionable item). Since there can be ties, mode() returns a Series not a single value. Use [0] to grab the first (most common) one. Example: ['Pop', 'Rock', 'Pop', 'Jazz'] → .mode()[0] = 'Pop'
🌍
How Spotify Wrapped works Every December, Spotify shows you your 'top 5 genres'. They do this by running value_counts() on all your streams from the year, then picking the top 5 entries. Your 'minutes listened' is .sum() of all track durations. The 'most streamed artist' is .idxmax() of a grouped count. You're learning the exact building blocks!
python
import pandas as pd
import matplotlib.pyplot as plt

data = {
    "Track":       ["Song A", "Song B", "Song C", "Song D"],
    "Genre":       ["Pop", "Rock", "Pop", "Jazz"],
    "Duration":    [210, 180, 200, 230],   # seconds
    "Danceability":[0.8, 0.6, 0.9, 0.5]
}
df = pd.DataFrame(data)

# Average duration in seconds
print("Avg Duration:", df["Duration"].mean())   # → 205.0

# Most popular genre (.mode() = most frequent value)
print("Top Genre:", df["Genre"].mode()[0])       # → "Pop"
# [0] because mode() returns a Series — grab first

# Count per genre
print(df["Genre"].value_counts())
# Pop     2
# Rock    1
# Jazz    1

# Bar chart of genre distribution
df["Genre"].value_counts().plot(kind="bar", title="Genre Distribution")
plt.ylabel("Count")

Pie charts and autopct Pie charts show proportions of a whole — best when you have 3–6 categories that add up to 100%. The autopct parameter adds text to each slice: %1.1f means 1 digit before decimal, 1 after (e.g., 40.0). The %% escapes to print a literal percent sign. Without autopct your slices show no values at all.

IMDb and BoxOffice Mojo IMDb and BoxOffice Mojo do exactly this analysis on their database of 500,000+ films. Their genre distribution pie charts, average rating by genre, and year-over-year trends all use Pandas groupby + matplotlib visualizations internally.

🤖 Machine Learning — Introduction
Machine Learning teaches computers to find patterns in data and make predictions — without being explicitly programmed for every case.
ML IS EVERYWHERE AROUND YOU
Gmail Spam Filter — classifies emails as spam/not-spam (classification). Google Maps ETA — predicts your arrival time (regression). ChatGPT — predicts the most likely next word given all previous words (a very fancy regression). Every one of these uses the same fundamental principles you're about to learn.
🏷️ Classification — "Which category is this?"

Output is a label / class. Examples: Is this email spam? Will this student pass? What dog breed is this? Is this tumor malignant?

Algorithms: Logistic Regression, Decision Tree, Random Forest, SVM, KNN

📈 Regression — "What number will this be?"

Output is a number. Examples: What will this house cost? How many units will we sell next month? What score will this student get?

Algorithms: Linear Regression, Ridge, Lasso, Polynomial Regression

🔄
The Universal ML Workflow — 5 steps every project follows:
  1. Collect & Clean Data — the dataset is your raw material
  2. Separate Features (X) and Target (y) — X = inputs, y = what you want to predict
  3. Create & Train the Model — model = Algorithm(); model.fit(X, y)
  4. Make Predictions — model.predict(new_data)
  5. Evaluate — accuracy_score (classification) or MAE/R² (regression)
💡
Golden rule: Features (X) must be numbers. Scikit-learn models only understand numbers. Text like "pass/fail" or "Brown/Black" must be converted BEFORE training. For text targets: LabelEncoder converts "pass" → 0, "fail" → 1. For text features with no order: pd.get_dummies() creates 0/1 columns. Always check df.dtypes before calling model.fit().

python
import pandas as pd
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score

data = {
    "student":     ["Sammy", "John", "Simran", "Tina"],
    "attendance":  ["60%",  "40%",  "20%",    "90%"],
    "assignments": [10, 5, 8, 8],
    "target":      ["pass", "pass", "fail", "pass"]
}
df = pd.DataFrame(data)

# Convert "60%" string → 60 (number)
df["attendance"] = df["attendance"].str.replace("%", "").astype(int)

# Features (X) and Target (y)
X = df[["attendance", "assignments"]]
y = df["target"]

# Train the model
model = LogisticRegression()
model.fit(X, y)

# Predict for a new student: 70% attendance, 5 assignments
prediction = model.predict([[70, 5]])
print("Prediction:", prediction[0])   # → "pass" or "fail"

# Accuracy on training data
y_pred = model.predict(X)
print("Accuracy:", accuracy_score(y, y_pred))

⚠️
One-Hot Encoding — the categorical data problem ML models only understand numbers. When you have a 'color' column with values 'Brown', 'Black', 'White', you can't just assign them 1, 2, 3 — the model would think White (3) is 'more color' than Brown (1), which makes no sense. pd.get_dummies() creates 3 separate columns: color_Brown, color_Black, color_White. Each is 0 or 1. No false ordering.
⚠️
The reindex() trick is critical When you apply get_dummies() to a single new dog with color 'White', you only get ONE column: color_White. But the model was trained with THREE columns: color_Black, color_Brown, color_White. The .reindex(columns=X.columns, fill_value=0) adds the missing columns with value 0. Without this, the model will throw an error.
🌍
Computer Vision and pet apps PetFinder.my and Microsoft's Custom Vision use multi-class classification (exactly like this, but with image features instead of weight/height) to identify dog breeds from photos. The largest breed classification models have 120+ classes (breeds). Google Lens uses a deep learning version of this very concept.
python
import pandas as pd
from sklearn.linear_model import LogisticRegression

data = {
    "weight": [30,25,28,20,35,22,40,18,32,15,25,10],
    "height": [60,55,58,50,65,52,70,45,68,40,56,38],
    "age":    [5, 4, 6, 3, 7, 2, 6, 4, 5, 3, 4, 2],
    "color":  ["Brown","Black","Black","White","Brown","White",
              "Black","Brown","Brown","White","Black","White"],
    "breed":  ["Labrador"]*3 + ["Poodle"]*3 + ["Husky"]*3 + ["Beagle"]*3
}
df = pd.DataFrame(data)

# Convert "color" text column → numeric dummy columns
df_dummies = pd.get_dummies(df, columns=["color"])
# Creates: color_Black, color_Brown, color_White (all 0 or 1)

X = df_dummies.drop("breed", axis=1)
y = df_dummies["breed"]

model = LogisticRegression(max_iter=2000)
# max_iter=2000: give model more steps to find the best answer
model.fit(X, y)

# Predict new dog: weight=25, height=55, age=4, color=White
new_dog = pd.DataFrame({"weight":[25],"height":[55],"age":[4],"color":["White"]})
new_dummy = pd.get_dummies(new_dog, columns=["color"])
# Align columns with training data
new_dummy = new_dummy.reindex(columns=X.columns, fill_value=0)

print("Predicted breed:", model.predict(new_dummy)[0])

Key Evaluation Metrics for Regression: MAE (Mean Absolute Error) — average distance between predicted and actual value, in the same unit as your data. If MAE = 15 for house prices in $1000s, your model is off by $15,000 on average. Lower is better.
R² Score (R-squared) — how much of the variance in y the model explains. 1.0 = perfect prediction. 0.0 = no better than predicting the average every time. 0.85 means the model explains 85% of the variation. Higher is better.
y = b₀ + b₁·x₁ + b₂·x₂ + ...  (Linear Regression finds the best values of b₀, b₁, b₂...)

python
import gradio as gr
import pandas as pd
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_wine

# Load built-in wine dataset (178 wines, 13 chemical features)
wine = load_wine()
df = pd.DataFrame(wine.data, columns=wine.feature_names)
df["target"] = wine.target   # 0, 1, or 2 (wine class)

X = df.drop("target", axis=1)
y = df["target"]

model = RandomForestClassifier()
model.fit(X, y)

def predict_wine(alcohol, malic_acid, ash, alcalinity_of_ash, magnesium,
                  total_phenols, flavanoids, nonflavanoid_phenols,
                  proanthocyanins, color_intensity, hue, od280, proline):
    inputs = [[alcohol, malic_acid, ash, alcalinity_of_ash, magnesium,
               total_phenols, flavanoids, nonflavanoid_phenols,
               proanthocyanins, color_intensity, hue, od280, proline]]
    pred = model.predict(inputs)[0]
    return f"Wine Class {pred} ({wine.target_names[pred]})"

# Build sliders for each feature automatically
with gr.Blocks() as demo:
    gr.Markdown("# 🍷 Wine Quality Classifier")
    inputs = [gr.Slider(label=col,
                        minimum=float(X[col].min()),
                        maximum=float(X[col].max()),
                        value=float(X[col].mean()))
              for col in X.columns]
    output = gr.Textbox(label="Predicted Wine Class")
    gr.Button("Predict").click(fn=predict_wine, inputs=inputs, outputs=output)

demo.launch()

How KMeans clustering works on images Every pixel is an [R, G, B] point in 3D color space. KMeans groups all pixels into 3 clusters by finding 3 center-points that minimize the total distance. After convergence, the center of the largest cluster is the dominant color — the color most pixels are 'near to'. Then we find the closest named color using Euclidean distance: √((R1-R2)² + (G1-G2)² + (B1-B2)²). This is unsupervised learning — no labels needed!

import gradio as gr
import numpy as np
from PIL import Image
from sklearn.cluster import KMeans

def color_name(rgb):
    """Find the closest named color to an RGB value"""
    colors = {
        "Red":(255,0,0), "Green":(0,255,0), "Blue":(0,0,255),
        "Yellow":(255,255,0), "Orange":(255,165,0),
        "Black":(0,0,0), "White":(255,255,255),
        "Gray":(128,128,128), "Purple":(128,0,128)
    }
    best, min_dist = "Unknown", float("inf")
    for name, value in colors.items():
        # Euclidean distance: sqrt((R1-R2)²+(G1-G2)²+(B1-B2)²)
        dist = np.linalg.norm(np.array(rgb) - np.array(value))
        if dist < min_dist:
            min_dist, best = dist, name
    return best

def predict_color(img):
    img  = img.resize((150, 150))           # shrink for speed
    pixels = np.array(img).reshape((-1, 3))  # (22500, 3) — all pixels

    kmeans = KMeans(n_clusters=3, n_init=10)
    kmeans.fit(pixels)
    # cluster_centers_ contains the average RGB of each cluster
    dominant = kmeans.cluster_centers_[0].astype(int)
    rgb = tuple(dominant)
    return f"🎯 Dominant Color: {color_name(rgb)}\n🔢 RGB: {rgb}"

with gr.Blocks() as demo:
    gr.Markdown("## 🎨 Color Prediction from Image")
    image  = gr.Image(type="pil", label="Upload Image")
    output = gr.Textbox(label="Result")
    gr.Button("Predict Color").click(predict_color, inputs=image, outputs=output)

demo.launch()

PIL Image modes explained PIL (Python Imaging Library) represents images internally using 'modes'. 'RGB' = 3 channels (Red, Green, Blue), each 0–255. 'L' = Luminance (grayscale), single channel 0–255. 'RGBA' = RGB + Alpha (transparency). Converting 'RGB' to 'L' averages the three color channels weighted by how the human eye perceives brightness (Green contributes most, Blue least).
