Classes & Objects
The building blocks of Object-Oriented Programming

🌍 What is OOP?

Object-Oriented Programming (OOP) is a way of writing code that mirrors the real world. Everything around you — a pen, a car, a person — is an object. OOP lets you model those real things in your code!

🌟 Real-Life Example

Pick up a pen from your desk. Did anything else in the room change? No! That's because the world is made of individual objects. Similarly, in OOP, you can change one object without affecting others — just like painting your car yellow doesn't change your neighbour's car!

🏗️ What is a Class?

A class is like a blueprint or a recipe. Just like a car blueprint tells you what every car will have (wheels, engine, seats), a class tells Python what every object of that type will have.

📦 What is an Object?

An object is the actual thing created from the blueprint. A BMW and a Toyota are both objects of the Car class. Each has the same structure, but different values!

🔑 Key Vocabulary

TermMeaningReal-world example
ClassBlueprint / TemplateCar blueprint
ObjectCreated from the classYour specific car (BMW)
AttributeProperty / Variable inside a classColor, brand, wheels
MethodFunction inside a classdrive(), brake(), honk()
__init__Constructor — runs when object is createdFactory assembly line

📐 Class vs Instance Attributes

  • Class attributes — same for ALL objects (e.g. all cars have 4 wheels)
  • Instance attributes — different for each object (e.g. each car has its own color)
📁 Project 1

🚗 The Car Class

Build a Car blueprint with attributes like color, brand, model, fuel, and price. Create multiple car objects and display their info.

1. car.py Python 
class Car:
    wheels = 4      # class attribute — same for ALL cars

    def __init__(self, color, brand, model, fuel, price):
        self.color = color        # instance attributes — different per car
        self.brand = brand
        self.model = model
        self.fuel  = fuel
        self.__price = price   # private variable (double underscore)

    def display_info(self):
        print("Color:", self.color)
        print("Brand:", self.brand)
        print("Model:", self.model)
        print("Fuel :", self.fuel)
        print("Wheels:", Car.wheels)   # access class attr with class name
        print("Price:", self.__price)    # access private inside class

# Create 3 objects (instances) of Car
c1 = Car("green", "Ford",       "Mustang", "Gasoline",  1130238)
c2 = Car("red",   "Toyota",     "Prius",   "Electric",  2372139)
c3 = Car("blue",  "Volkswagen", "Golf",    "Diesel",    2873219)

c1.display_info()
c2.display_info()
c3.display_info()

# Change an instance attribute of only c1
c1.color = "yellow"
c1.display_info()   # c2 and c3 colors are unchanged!

# Change the class attribute for ALL objects
Car.wheels = 5       # now ALL cars show 5 wheels

🔍 Line-by-Line Explanation

  • class Car: — declares the Car blueprint
  • wheels = 4 — all cars have 4 wheels by default (class attribute)
  • def __init__(self, ...) — this runs automatically when you create a Car object. It's like the factory setting up each car on the assembly line
  • self.color = color — stores the color for THIS specific car
  • self.__price — private variable! The double underscore means it can only be accessed inside the class (like a car's internal cost that you don't share)
  • c1 = Car(...) — creates an actual car object!
💡 Tip: Class names in Python should always start with a Capital Letter (Car, Person, Dog). This is a standard coding convention!
⚠️ Private Variables: Variables starting with __ (double underscore) are private — you can't access them from outside the class using c1.__price. This is called Encapsulation!

Special methods that belong to the class, not individual objects

📌 What is a Class Method?

A class method belongs to the class itself, not to any specific object. It uses @classmethod decorator and takes cls (the class) as the first parameter instead of self.


  • Uses @classmethod decorator above the function
  • First parameter is cls (stands for "class")
  • Can only access class attributes, NOT instance attributes
  • Can be called without creating an object: ClassName.method()
  • Can create and return a new object of the class
🌟 Real-Life Example

Imagine a school attendance counter. No matter which student you ask, the total number of students enrolled is the same — it belongs to the school (class), not any individual student. A class method lets you count how many Student objects have been created!

Build a full deck of 52 playing cards using two classes: Card and Deck. Deck contains Card objects as attributes — objects inside objects!

🗂️ Structure Overview

  • card.py — defines a single Card (suit + value, with validation)
  • deck.py — defines a Deck that holds 52 Card objects
  • main.py — the game menu to display, shuffle, and deal cards
card.py — Single Card Python 
class Card:
    def __init__(self, suit, value):
        self.suit  = ""
        self.value = ""
        self.setSuit(suit)
        self.setValue(value)

    def setSuit(self, s):
        if s in ["diamonds", "hearts", "clubs", "spades"]:
            self.suit = s
        else:
            print("Invalid suit!")

    def setValue(self, v):
        valid = ['A','K','Q','J','2','3','4','5','6','7','8','9','10']
        if v in valid:
            self.value = v
        else:
            print("Invalid value!")

    def displayCard(self):
        print(f"{self.value} of {self.suit}")
deck.py — 52 Cards using Class Method Python 
from card import Card
import random

class Deck:
    def __init__(self):
        self.cards = Deck.populate()  # use class method to fill the deck

    @classmethod
    def populate(cls):
        suits  = ["diamonds", "hearts", "spades", "clubs"]
        values = ['A','K','Q','J','2','3','4','5','6','7','8','9','10']
        cards  = []
        for suit in suits:
            for value in values:
                cards.append(Card(suit, value))   # Card objects inside Deck!
        return cards   # 4 suits × 13 values = 52 cards

    def displayDeck(self):
        for card in self.cards:
            card.displayCard()
        print("Total:", len(self.cards))

    def shuffleDeck(self):
        random.shuffle(self.cards)
        self.displayDeck()

    def deal(self):
        i = random.randint(0, len(self.cards) - 1)
        c = self.cards.pop(i)
        print("Card dealt:")
        c.displayCard()
        print("Remaining:", len(self.cards))
main.py — Interactive Menu Python 
from card import Card
from deck import Deck

d1 = Deck()

while True:
    print("""
    1. Display deck
    2. Shuffle deck
    3. Deal a card
    4. Exit
    """)
    choice = int(input("Enter 1-4: "))
    if   choice == 1: d1.displayDeck()
    elif choice == 2: d1.shuffleDeck()
    elif choice == 3: d1.deal()
    elif choice == 4: break
    else: print("Wrong choice")
🌟 Real-Life Connection

This is exactly how a real card game app is built! Card and Deck are separate classes. The Deck holds Card objects as its attribute. Class method populate() acts like a factory that mass-produces all 52 cards automatically.

Inheritance
Reuse code by building new classes on top of existing ones

🧬 What is Inheritance?

Inheritance means a child class can get (inherit) all the properties and methods of a parent class, and then add its own extra features. This is code reuse at its best!

🌟 Real-Life Example

Think about animals. All animals have a name and an age. But a Dog also barks, and a Bird also flies. Instead of repeating name and age in every animal class, we put common stuff in the Animal parent class, and Dog/Bird inherit from it. Just like how you inherit traits (eye color, height) from your parents!

👪 Parent / Child Classes

  • Parent class (base class) = the original class with common features
  • Child class (derived class) = inherits from parent + adds new features
  • Syntax: class Child(Parent):

🔗 super().__init__()

  • super() calls the parent class
  • Use it in the child's __init__ to set up the parent's attributes first
  • Prevents repeating parent code in every child class
📁 Example 1

🦁 Animals — Single & Hierarchical Inheritance

animal2.py — Inheritance with super() Python 
class Animal:                                 # Parent class
    def __init__(self, name, age, legs=4):    # default value for legs
        self.name = name
        self.age  = age
        self.legs = legs

    def print_info(self):
        print("Name:", self.name)
        print("Age :", self.age)
        print("Legs:", self.legs)

class Mammal(Animal):                        # Child class
    def __init__(self, name, age, kids):
        super().__init__(name, age, 4)          # call Animal's __init__
        self.kids = kids                        # extra attribute for Mammal

    def print_running(self):
        print("The", self.name, "is running")

# Animal object
a = Animal("Lion", 10, 4)
a.print_info()

# Mammal object (inherits ALL Animal methods)
m = Mammal("Monkey", "15", 5)
m.print_info()       # inherited from Animal!
m.print_running()    # Mammal's own method
print(m.kids)        # Mammal's own attribute

class Character:

paladin.cast_spell("Lightning Strike")  # from Mage

Method Overriding
Child class redefines a parent class method to do something different

✏️ What is Method Overriding?

When a child class has a method with the same name as the parent class, the child's version overrides (replaces) the parent's. Python will use the child's version when called on a child object.

🌟 Real-Life Example

Think of a bank's interest rate. Every bank has an interest rate method, but SBI gives 7%, ICICI gives 8%, and a default Bank gives 10%. Each bank overrides the general rate with its own specific rate. Same method name, different result!

📁 Example — Birds

🐦 Birds Override flight()

3.Bird.py Python 
class Bird:
    def intro(self):
        print("There are many types of birds.")
    def flight(self):
        print("Most birds can fly but some cannot.")

class Sparrow(Bird):
    def flight(self):           # OVERRIDES Bird's flight()
        print("Sparrows can fly!")

class Ostrich(Bird):
    def flight(self):           # OVERRIDES Bird's flight()
        print("Ostriches cannot fly.")

b = Bird();    b.intro(); b.flight()    # "Most birds can fly..."
s = Sparrow(); s.intro(); s.flight()   # "Sparrows can fly!"
o = Ostrich(); o.intro(); o.flight()   # "Ostriches cannot fly."
📁 Project 8 — MRO & Vehicles

🚗🚛 Vehicle Hierarchy with MRO

Multiple levels of inheritance (Vehicle → Car → SportsCar) and Method Resolution Order!

2. vehicle.py Python 
class Vehicle:
    def __init__(self, price, color):
        self.color = color
        self.price = price
        self.gas   = 0

    def fillUpTank(self):  self.gas = 100
    def move(self):
        while self.gas > 0:
            print("Moving...")
            self.gas -= 1
        return "Gas over!"

    def display_info(self):  # overridden by children
        print(self.price, self.color)

class Car(Vehicle):
    def __init__(self, price, color, speed):
        super().__init__(price, color)
        self.speed = speed
    def display_info(self):    # OVERRIDES Vehicle's display_info
        print(self.price, self.color, self.speed)

class SportsCar(Car):        # multilevel: SportsCar→Car→Vehicle
    def __init__(self, price, color, speed, radiators):
        super().__init__(price, color, speed)
        self.radiators = radiators
    # No display_info here — so it uses Car's version (MRO!)

Bugatti = SportsCar(10000, "red", 400, 12)
Bugatti.fillUpTank()
print(Bugatti.move())
Bugatti.display_info()     # Uses Car's display_info (MRO)

# isinstance checks
print(isinstance(Bugatti, SportsCar))  # True
print(isinstance(Bugatti, Car))        # True!
print(isinstance(Bugatti, Vehicle))    # True!
💡 MRO (Method Resolution Order): When Python looks for a method, it searches in this order: current class → parent class → grandparent class → ... This is why SportsCar uses Car's display_info — it goes up the chain until it finds one!

Polymorphism
poly = many, morphism = forms — one name, many behaviors!

🦎 What is Polymorphism?

Polymorphism means the same action can behave differently on different objects. Like the word "run" — a person runs on legs, a car runs on fuel, a program runs on a computer!

🔢 Operator Overloading

Making operators like +, -, < work with your own objects.

4 + 5 = 9 (numbers)
"4" + "5" = "45" (strings)
Same + operator, different behavior!

🔁 Method Overloading

Same method name works with different numbers of arguments. Python uses default parameters to achieve this.

📁 Concept 1

➕ Operator Overloading

operator_overloading.py Python 
class Vehicle:
    def __init__(self, fare):
        self.fare = fare

    def __add__(self, other):       # overloads the + operator
        return self.fare + other.fare

    def __lt__(self, other):        # overloads the < operator
        return self.fare < other.fare

bus = Vehicle(20)
car = Vehicle(30)

total_fare = bus + car      # calls __add__
print(total_fare)           # 50

compare = bus < car         # calls __lt__
print(compare)              # True

📐 Shapes — Polymorphism with area()

shape.py, rectangle.py, square.py, main.py Python 
# shape.py — Parent
class Shape:
    def __init__(self, n):
        self.num_sides = n
    def info(self):
        print("2D shape with", self.num_sides, "sides")
    def area(self):
        print("Calculating area...")
    def perimeter(self):
        print("Calculating perimeter...")

# rectangle.py — Child
from shape import Shape
class Rectangle(Shape):
    def __init__(self, side1, side2):
        super().__init__(4)
        self.length = side1
        self.width  = side2
    def area(self):
        super().area()
        return self.length * self.width
    def perimeter(self):
        super().perimeter()
        return 2 * (self.length + self.width)

# square.py — Child
from shape import Shape
class Square(Shape):
    def __init__(self, side_len):
        super().__init__(4)
        self.side_len = side_len
    def area(self):
        super().area()
        return self.side_len ** 2
    def perimeter(self):
        super().perimeter()
        return 4 * self.side_len

# main.py — Using Shapes
from rectangle import Rectangle
from square import Square

r = Rectangle(10, 5)
r.info()                 # from Shape
print(r.area())          # 50
print(r.perimeter())     # 30

s = Square(20)
s.info()
print(s.area())          # 400
print(s.perimeter())     # 80

Both a Rectangle and a Square have an area() method — but they calculate it differently! This is polymorphism: same action (find area), different shapes, different formulas. A math teacher says "find the area" to all students — but the formula depends on the shape!

Abstraction
Hide the "how" and show only the "what" — simplicity through design

🔲 What is Abstraction?

Abstraction means hiding complex details and showing only what's necessary. An abstract class is a blueprint that cannot be used to create objects directly. It forces child classes to provide their own implementation of certain methods.

🌟 Real-Life Examples

When you drive a car, you press the accelerator — you don't know how the engine works internally! The car abstracts the complexity. Similarly, when you use a remote control, you press "Volume Up" — you don't care about the electronics inside. Abstract classes work the same way: they define what must be done, but not how.

📋 Rules of Abstract Classes

  • Import ABC and abstractmethod from the abc module
  • Class inherits from ABC
  • Abstract methods use @abstractmethod decorator
  • You cannot create an object of an abstract class
  • Child class must implement all abstract methods
  • If child doesn't implement, it also becomes abstract!

🔑 Key Concept

Abstract class = Contract! It says: "Every subclass MUST have these methods, but you decide HOW they work."


Example: A Shape class says every shape MUST have area() and perimeter(). But Circle, Rectangle, Triangle each calculate them differently.

📁 Project 12

🚗 Abstract Car — Mileage

car.py — Abstract Car Python 
from abc import ABC, abstractmethod

class Car(ABC):       # abstract class — inherits from ABC
    @abstractmethod

# shape.py — Abstract Base Class
from abc import ABC, abstractmethod

class Shape(ABC):
    def __init__(self, n):
        self.num_sides = n
    def info(self):   # NOT abstract — has implementation
        print("2D shape with", self.num_sides, "sides")
    @abstractmethod
    def area(self): pass       # MUST be overridden
    @abstractmethod
    def perimeter(self): pass  # MUST be overridden

# rectangle.py
from shape import Shape
class Rectangle(Shape):
    def __init__(self, side1, side2):
        super().__init__(4)
        self.length = side1
        self.width  = side2
    def area(self):
        return self.length * self.width
    def perimeter(self):
        return 2 * (self.length + self.width)
    def hint(self):
        self.info()
        print("My opposite sides are equal!")

import turtle, random

class Flower(turtle.Turtle):   # Flower IS a Turtle — inheritance!
    colors = ["#a31b03", "#cbfaa7", "#3c3457", "#ed5fc2",
              "medium purple", "peru", "gold", "deep pink",
              "tomato", "magenta"]   # class attribute

    def __init__(self):
        turtle.Turtle.__init__(self)           # setup turtle
        self.color(random.choice(Flower.colors))  # random color
        self.radius   = random.randint(5, 20)     # random size
        self.pensize(random.randint(3, 6))
        self.speed(0)

    def drawFlower(self):
        for i in range(6):
            self.circle(self.radius)   # draw petal
            self.right(60)             # rotate

    def changeLocation(self, x, y):
        self.pu()        # pen up (don't draw while moving)
        self.goto(x, y)  # move to position
        self.pd()        # pen down (start drawing again)

s = turtle.Screen()

s.bgcolor("lavender")
s.title("Colorful Flower Garden")

while True:                              # keep drawing flowers!
    f1 = Flower()
    f1.changeLocation(
        random.randint(-400, 400),       # random x
        random.randint(-300, 300)        # random y
    )
    f1.drawFlower()
🌟 Why OOP Here?

Each Flower object is independent — its own color, size, and position. Without OOP, you'd need separate variables for each flower. With OOP, you just create a new Flower() object and it sets itself up automatically!

3. car.py — Moving Car Animation Python 
import turtle, time, random

class Car(turtle.Turtle):
    def __init__(self):
        super().__init__()
        self.setx(-400)          # start from left
        self.hideturtle()
        self.drawCar()
        self.color(random.choice(["red","green","blue","yellow"]))

    def drawCar(self):     # draws the car shape using turtle movements
        self.clear()
        self.begin_fill()
        self.right(45); self.fd(50); self.right(135); self.fd(100)
        self.right(135); self.fd(50); self.right(45); self.fd(30)
        # ... continues drawing body, wheels ...
        self.dot(50, "black")   # draw wheels as dots
        self.end_fill()

    def moveCar(self, amt):
        self.clear()
        self.pu()
        self.seth(0)              # face East (0=East, 90=North)
        self.checkLocation()
        self.fd(amt)
        self.pd()
        self.drawCar()

    def checkLocation(self):
        if self.xcor() >= 400:   # if at right edge, go back to left
            self.setx(-400)
            self.color(random.choice(["red","green","blue","yellow"]))

s = turtle.Screen()
s.setup(800, 500)
s.tracer(False)   # turn off auto-drawing (faster)
c = Car()

while True:
    c.moveCar(5)    # move 5 pixels at a time
    s.update()      # refresh screen
    time.sleep(0.05)

📁 File 4 — Main Game Loop

🎮 main.py — Putting It All Together

main.py — Complete Game Python 
import turtle
from player import Player
from alien  import Alien
from flower import Flower
import os, winsound

# ─── Setup Screen ───────────────────────────
scr = turtle.Screen()
scr.title("Space Invaders")
scr.bgpic(r"space invader images/dp.gif")
scr.setup(800, 500)

# ─── Create Objects ──────────────────────────
p = Player()    # spaceship
b = Flower()    # bullet (starts hidden)

aliens = []
for i in range(7):
    aliens.append(Alien())   # 7 alien objects!

# ─── Shoot Function ──────────────────────────
def shoot():
    if b.state == "Ready":
        b.state = "shoot"
        winsound.PlaySound(r"space invader images/explosion.wav",
                           winsound.SND_ASYNC)
        b.goto(p.xcor(), p.ycor() + 20)  # start at player position
        b.showturtle()

# ─── Key Bindings ─────────────────────────────
turtle.listen()
turtle.onkey(p.move_left,  "Left")
turtle.onkey(p.move_right, "Right")
turtle.onkey(shoot,        "space")

# ─── Collision Functions ─────────────────────
def collision_flower_alien(alien):
    return b.distance(alien) < 25   # True if bullet hits alien

def collision_player_alien(alien):
    return p.distance(alien) < 25   # True if alien hits player

# ─── Score Display ───────────────────────────
score = 0
s = turtle.Turtle()
s.pu(); s.goto(200, 200); s.color("white"); s.ht()

# ─── Main Game Loop ───────────────────────────
while True:
    for alien in aliens:
        # Move alien sideways
        alien.setx(alien.xcor() + alien.speedamt)

        # If alien hits right wall, all aliens go down + reverse
        if alien.xcor() > 400:
            for j in aliens: j.sety(j.ycor() - 25)
            alien.speedamt *= -1

        # If alien hits left wall, same thing
        if alien.xcor() < -400:
            for j in aliens: j.sety(j.ycor() - 25)
            alien.speedamt *= -1

        # Bullet hits alien!
        if collision_flower_alien(alien):
            aliens.remove(alien)
            alien.ht()
            score += 1
            s.clear()
            s.write(score, font=("Arial", 40))

        # Alien hits player — GAME OVER
        if collision_player_alien(alien):
            for i in aliens: i.ht()
            s.clear(); s.goto(0, 0)
            s.write("Game Over", align="center", font=("Arial", 40))
            break

    # Move bullet upward
    b.sety(b.ycor() + b.speedamt)

    # If bullet goes off screen, reset it
    if b.ycor() > 300:
        b.hideturtle()
        b.state = "Ready"

turtle.done()

📋 How the Game Works — Step by Step

  1. 1
    Setup: Screen is created, background set to space image. Player, 7 aliens, and 1 flower-bullet are created.
  2. 2
    Key Bindings: Left arrow → move_left(), Right arrow → move_right(), Space → shoot()
  3. 3
    Alien Movement: Each alien moves sideways. When it hits the wall, ALL aliens drop down and reverse direction (like real Space Invaders!).
  4. 4
    Bullet: Starts as "Ready". When Space is pressed, it appears at player position and flies upward at speedamt per frame.
  5. 5
    Collision: If bullet gets within 25px of an alien → alien is removed, score +1. If alien gets within 25px of player → GAME OVER!
  6. 6
    Score: A separate turtle object displays the score at (200, 200) in white text.
